# i18n (English + Brazilian Portuguese) — Implementation Plan

> **For agentic workers:** This is an orchestration plan. The user's main Claude thread is the **orchestrator**. Each Task below corresponds to a subagent dispatch — the orchestrator composes the prompt, runs `Agent`, verifies output, and commits. Do NOT execute these tasks directly in the main thread; the work is partitioned across subagents to keep context lean.

**Goal:** Ship a bilingual UI shell (English + PT-BR) for the Gnars DAO website using `next-intl`, subpath routing (`/` for English, `/pt-br/...` for Portuguese), and a single PR delivered through coordinated subagent dispatches in the current worktree.

**Architecture:** `next-intl` with `localePrefix: "as-needed"`, per-namespace JSON files merged at runtime in `i18n/request.ts`, `[locale]` dynamic segment wrapping all UI routes, `next-intl` middleware merged into existing `src/proxy.ts` alongside the markdown-rewrite logic. Translation generated by a translation agent following a tone brief; extraction across 9 disjoint slices in 3 batches.

**Tech Stack:** Next.js 16.2.4 (App Router, `proxy.ts` convention), React 19.2.5 (React Compiler ON), TypeScript, `next-intl` (new dep), vitest (unit tests for proxy locale-stripping).

**Worktree:** `.claude/worktrees/quirky-cray-b26c80` on branch `claude/quirky-cray-b26c80`. Single PR to `main`.

**Spec:** [docs/superpowers/specs/2026-05-11-i18n-en-ptbr-design.md](../specs/2026-05-11-i18n-en-ptbr-design.md)

```ts
import { NextRequest, NextResponse } from "next/server";
import createIntlMiddleware from "next-intl/middleware";
import { routing } from "@/i18n/routing";

const intlProxy = createIntlMiddleware(routing);

const MARKDOWN_PATHS = ["/", "/proposals"];
const MARKDOWN_DYNAMIC_PATTERNS = [/^\/proposals\/[^/]+$/];

function stripLocale(pathname: string): string {
  for (const locale of routing.locales) {
    if (locale === routing.defaultLocale) continue;
    if (pathname === `/${locale}`) return "/";
    if (pathname.startsWith(`/${locale}/`)) return pathname.slice(`/${locale}`.length);
  }
  return pathname;
}

function isMarkdownPath(pathname: string): boolean {
  const stripped = stripLocale(pathname);
  if (MARKDOWN_PATHS.includes(stripped)) return true;
  return MARKDOWN_DYNAMIC_PATTERNS.some((p) => p.test(stripped));
}

export function proxy(request: NextRequest): NextResponse {
  const accept = request.headers.get("accept") ?? "";
  if (accept.includes("text/markdown") && isMarkdownPath(request.nextUrl.pathname)) {
    const stripped = stripLocale(request.nextUrl.pathname);
    const mdPath = stripped === "/" ? "/md" : `/md${stripped}`;
    return NextResponse.rewrite(new URL(mdPath, request.url));
  }
  return intlProxy(request);
}

export const config = {
  matcher: ["/((?!api|_next|_vercel|md|.*\\..*).*)"],
};
```

HOW TO USE t() (NEXT.JS 16 + NEXT-INTL):
- File starts with `"use client"`?
  ```ts
  import { useTranslations } from "next-intl";
  // inside component
  const t = useTranslations("<your-namespace>");
  ```
- Server component (no `"use client"`)?
  ```ts
  import { getTranslations } from "next-intl/server";
  // inside async function/component
  const t = await getTranslations("<your-namespace>");
  ```
- Server page/layout under `[locale]` that uses translations? Call `setRequestLocale(locale)` at the top before any t() call:
  ```ts
  import { setRequestLocale } from "next-intl/server";
  // ...
  const { locale } = await params;
  setRequestLocale(locale);
  ```
- Interpolation: `t("greeting", { name: user })` with key `"Hello, {name}"`.
- Plurals (ICU): `t("count", { count: n })` with key `"{count, plural, =0 {No items} one {# item} other {# items}}"`.
- Rich text (links/bold inside translation): `t.rich("term", { link: (chunks) => <a href="...">{chunks}</a> })`.
- Markup-as-text (when you don't need element renaming): use a separate key per part instead of t.rich for simplicity.
